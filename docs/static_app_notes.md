# Static App Notes

This application is intentionally simple:

- No package manager.
- No build process.
- No database.

It's a static HTML/CSS/JS site that can be hosted anywhere (GitHub Pages,
Netlify, S3, etc.) — but it is **not** fully self-contained: recipe data is
fetched from a Google Sheet via an Apps Script Web App on every page load
(see [`docs/community_recipes.md`](community_recipes.md)), so an internet
connection is required and `file://` won't work.

On load, `assets/app.js` shows a loading spinner, calls `loadRecipeData()`
to fetch all `Approved` recipes from `RECIPES_API_URL`, and runs
`populateRecipeData()` to fill in the recipe list, sides, header images,
cooking instructions, and per-recipe costs before rendering anything. If the
fetch fails or returns no recipes, the app shows a full-page error screen
instead.

`assets/recipe-images/*-wide.webp` (recipe header images) and
`assets/recipe-images/*-thumb.webp` (Recipe Finder card thumbnails, square
crops derived from the wide images via `scripts/generate_recipe_thumbnails.mjs`)
remain static files bundled with the app, along with
`data/walmart_item_prices.js` (generic item price reference data for the
shopping list).

If you add a new `<id>-wide.webp` header image (e.g. via
`scripts/generate_missing_recipe_images.mjs`), run
`node scripts/generate_recipe_thumbnails.mjs` afterward to generate the
matching `<id>-thumb.webp`.

To publish it, upload the full folder to any static host. Keep the folder
structure intact.
